# Introduction

`jsonapi_core` is a typed [JSON:API v1.1](https://jsonapi.org/format/) serialization
library for Rust. It gives you a complete type model for JSON:API documents — resources,
relationships, links, errors — together with a derive macro that handles the envelope
format so you work with plain Rust structs.

## What's in the box

| Capability | Where it lives |
|------------|----------------|
| Full type model — `Document`, `Resource`, `Relationship`, `Link`, `ApiError`, etc. | `jsonapi_core::model` (re-exported at the crate root) |
| `#[derive(JsonApi)]` macro mapping structs to JSON:API resource envelopes | `jsonapi_core_derive` (re-exported as `jsonapi_core::JsonApi` when the `derive` feature is on) |
| Fuzzy deserialization that accepts camelCase, snake\_case, kebab-case, and PascalCase | Generated by the derive macro |
| Lookup table for `included` resources with typed and dynamic getters | `jsonapi_core::Registry` |
| Recursive resolver producing flattened, kitsu-core-style output with cycle detection | `Registry::resolve` |
| Static type metadata for include-path validation | `jsonapi_core::TypeRegistry` |
| Sparse fieldset filtering — typed and dynamic | `jsonapi_core::FieldsetConfig` |
| JSON:API-aware query string builder with RFC 3986 percent-encoding | `jsonapi_core::QueryBuilder` |
| Media-type parsing and content negotiation (`ext`, `profile`) | `jsonapi_core::media_type` |
| Member name validation per JSON:API 1.1 rules | `jsonapi_core::validation` |
| Atomic Operations extension (feature-gated) | `jsonapi_core::atomic` |

## Who this guide is for

This book assumes you have read or skimmed the [JSON:API specification](https://jsonapi.org/format/)
and are comfortable with serde. You don't need to be a JSON:API expert — most chapters
explain the relevant slice of the spec as they go.

## How to read this book

The chapters build on each other in order, but each one is self-contained:

1. **Getting Started** walks through installing the crate and serializing/deserializing
   your first resource.
2. **Core Concepts** covers the fundamental types: `Document`, `Resource`, the derive
   macro, relationships, and the `Registry`.
3. **Working with Requests and Responses** covers the request/response toolkit:
   the query builder, sparse fieldsets, include validation, content negotiation,
   and error handling.
4. **Extensions** covers the optional Atomic Operations extension.
5. **Reference** is a flat lookup for the derive macro's full attribute syntax,
   feature flags, and a cookbook of small recipes.

## A note on stability

`jsonapi_core` is at version **0.1.1** and the public API is intentionally
small but evolving. All public enums and structs that may grow new variants
or fields are marked `#[non_exhaustive]`, so additions won't be breaking
changes — but minor signatures may still shift before 1.0.
